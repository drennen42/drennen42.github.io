Ruby Classes

In all programming languages, classes help developers from having to type repetitive code. Creating a class allows a developer to create an object of that class that automatically inherits all characteristics of that class. When a new instance of a class is created, all inherited variabled and methos that come along with that object are called 'instance variables' and 'instance methods'. Changes made to the instance variables and methods only effect themselves and not the class that they were formed from.

One classic basic examples that is used when learning to create classes, instance variables, and instance methods of that class is the vehicle class example.

In the vehicle class example, you create a class called 'vehicle' that has variables such as 'num_wheels', 'num_doors', 'color', 'max_speed', and any other broad characteristics of a vehicle. Along with these variables, there are methods such as 'beep', 'go', and 'stop'.

It is necessary to initialize the variables within the class so that new objects of the class can use them.

Once the class is created, you can create as many instances of the class as you'd like, each instance of the class will contain all the previously initialized variables and methods of the vehicle class. For example, if I had Ford Mustang and a Toyota Camry, I would create two instances of the vehicle class like so:

  • ford_mustang = Vehicle.new(4,4,silver,190)
  • toyota_camry = Vehicle.new(4,4,black,150)

We now have two vehicle objects, both with 4 wheels and 4 doors, the Mustang is silver and has a max speed of 190 while the Camry is black with a max speed of 150.

Both of these vehicles can beep, go, and stop.
