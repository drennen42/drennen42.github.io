SQL Injections

Why would anyone want to do a SQL injection? A SQL injection is something that a hacker does to get access to all of the data stored in a website's backend database. The data can be user id's, passwords, banking information, account #s, just about anything. Much of the data the hacker is seeking is personal information and can be very harmful.

How are SQL injections performed? Say you're about to login to a website like Facebook and you're looking at the login fields 'user name' and 'password'. The site probably has a database of all of its users' user names and passwords that is cross checks the users input for validation. The website's [pseudo]code may be soemthing like:

>Grab username from user_name text field
>SELECT account FROM tbl_accounts WHERE user_name=username

So what if in the user name text field you entered in 'NOT NULL'? Then the site's code would pull all accounts from tbl_accounts where the user_name is not null!!

How to prevent SQL attacks? FIrewalls and other intrusion detection mechanisms provide little defense against full-scale web attacks. The best way to prevent SQL injection attacks is to use prepared statements, using stored procedures, and escaping all user supplied input. There are many resources online where you can find examples of how to use prepared statements and other suggestions on how to prevent SQL injections.
